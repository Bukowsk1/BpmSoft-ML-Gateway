# BPMSoft ML Router

Это интеграционный ML API роутер с демо дашбордом для маршрутизации запросов прогнозирования спроса в системе BPMSoft

Этот сервис выступает единой точкой входа. Он реализует концепцию AutoML: прогоняет данные через несколько доступных моделей под капотом, сверяется с офлайн метриками и возвращает лучший результат

## Архитектура и модели

В роутере интегрированы две модели прогнозирования спроса:

1. **Karina Model:** Основная продакшн модель, построенная на LightGBM и XGBoost
2. **RL Demand:** Экспериментальный RL агент (Stable-Baselines3 / PPO). Она берет предсказания базовых средних и корректирует их на основе признаков

### Как выбирается победитель

Роутер не выбирает модель вслепую. Для каждого входящего запроса он анализирует исторические офлайн метрики по следущзей логике:

1. **SKU:** Если RL-агент показал прирост качества конкретно на этом товаре, то запрос отдается ему
2. **Сегмент:** Если истории по SKU нет, роутер смотрит на уровень сегмента, например `Milk-Seg2`
3. **Категория:** Если данных по сегменту не хватает, оценивается категория целиком.
4. **Fallback:** Если RL модель показывает ухудшение метрик на любом из этапов проверки, роутер не рискует и мгновенно переключается на стабильную модель Карины

Это обеспечивает подход без риска просадки бизнес метрик. CRM всегда получает не только итоговую цифру, но и текстовое обоснование, почему была выбрана именно эта модель

## Структура проекта

* `Backend/app/` — Исходный код микросервиса FastAPI (эндпоинты, схемы валидации, слои бизнес логики).
* `Backend/app/services/demand_router.py` — Главный модуль, содержащий логику AutoML выбора.
* `Backend/artifacts/` — Скомпилированные веса моделей и CSV таблицы с предрассчитанными офлайн метриками
* `Backend/dashboard.py` — Демонстрационный дашборд
* `Karina/` — Исходный код и артефакты основной модели (Карина)
* `.env.example` — Шаблон переменных окружения

## Docker


1. 
   ```bash
   cd Backend
   cp .env.example .env
   ```
2. 
   ```bash
   docker compose up --build
   ```
3. Сервисы станут доступны по адресам:
   * API (Swagger UI): http://localhost:8000/docs
   * Streamlit Dashboard: http://localhost:8501

## Локальный запуск 

Можно поднять сервисы напрямую. Требуется `Python 3.11`

1. 
   ```bash
   cd Backend
   python3.11 -m pip install -r requirements.txt
   ```
2. Запуск апи в первом терминале:
   ```bash
   make api
   ```
3. Запуск дашборда во втором терминале:
   ```bash
   make dashboard
   ```

Для локального запуска LightGBM и PyTorch потребуется библиотека `libomp`. Конфликты потоков OpenMP между этими библиотеками уже решены на уровне конфигурации проекта (файл `runtime_checks.py`)

## Тестирование

Сервис покрыт юнит тестами, проверяющими логику маршрутизации и валидацию контрактов:

```bash
cd Backend
python3.11 -m pytest tests -v
```
